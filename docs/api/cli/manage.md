# Resource Management

The CiteKit CLI provides utilities for exploring, validating, and debugging your local resource index.

## `citekit list`

Lists all resources currently stored in the `.resource_maps/` directory.

### Usage
```bash
python -m citekit.cli list [RESOURCE_ID]
```

### Modes
1.  **Global List**: If `resource_id` is omitted, returns a table of all ingested resources, their types, and titles.
2.  **Detail List**: If a `resource_id` is provided, returns the **Full Semantic Tree** of that resource, including all nested section IDs.

---

## `citekit check-map`

Validates a JSON file against CiteKit's strict Pydantic models. Use this to verify maps generated by custom adapters or editors.

### Usage
```bash
python -m citekit.cli check-map path/to/map.json
```

**Performs**:
-   **JSON Syntax**: Checks for parsing errors.
-   **Type Safety**: Ensures fields like `resource_id`, `nodes`, and `location` meet CiteKit specs.
-   **Cross-Checks**: Warns if map type (e.g. `video`) doesn't match location types (e.g. `pages`).

---

## `citekit structure`

Returns the raw JSON content of a resource map.

### Usage
```bash
python -m citekit.cli structure <resource_id>
```
**Why?**: Pipe this output into other tools like `jq` or save it for debugging.
