Local AI Resource Mapping Made Simple

Let AI agents access specific parts of files—PDF pages, video clips, audio segments—without sending entire documents to the cloud.

Get Started View on PyPI 
from citekit import CiteKitClient

client = CiteKitClient()

# Ingest a file
map = await client.ingest("research.pdf")

# Resolve specific content
evidence = await client.resolve("research.intro.definition")
print(evidence.output_path)  # mini-PDF with just that section
import { CiteKitClient } from 'citekit';

const client = new CiteKitClient();

// Ingest a file
const map = await client.ingest("research.pdf");

// Resolve specific content
const evidence = await client.resolve("research.intro.definition");
console.log(evidence.output_path);  // mini-PDF with just that section

Why CiteKit?

Local-First

All file processing happens on your machine. No cloud storage. Your data stays private.

Precise Extraction

Get exact PDF pages, video clips, or image crops—not generic summaries.

AI-Powered

Uses LLMs to generate structured maps of your content automatically.

Multi-Modal

Supports PDFs, videos, audio, and images out of the box.

MCP Compatible

Works with Claude, Cline, and other MCP-compatible AI agents.

Zero Config

Install and start using in seconds. No complex setup required.

Quick Start

Installation

Python

pip install citekit

JavaScript

npm install citekit

Basic Usage

1

Set your API key

export GEMINI_API_KEY=your_key_here
2

Ingest a file

citekit ingest my_document.pdf
3

List available content

citekit list
4

Resolve specific content

citekit resolve my_document.section1.intro
